# Pulse

Personal command center CLI that answers **"what should I pay attention to right now?"**

Pulse combines morning briefings, project health monitoring, and AI service cost tracking into a single Go binary. Data collection runs via cron; reads are instant from a local SQLite database.

## Quick Start

```bash
# Install
go install github.com/xcoleman/pulse/cmd/pulse@latest

# Generate config
pulse config init

# Edit config to set your project scan directories
# ~/.config/pulse/config.yaml

# Collect data
pulse sync

# View your briefing
pulse
```

## Commands

| Command | Description |
|---------|-------------|
| `pulse` | Print morning briefing to stdout |
| `pulse sync` | Collect data from all enabled sources |
| `pulse sync --only git` | Run a single collector |
| `pulse tui` | Launch interactive Bubble Tea dashboard |
| `pulse obsidian` | Append briefing to today's Obsidian daily note |
| `pulse costs` | Print cost summary |
| `pulse costs --service claude` | Filter costs to a specific service |
| `pulse costs --period 7d` | Change the cost lookback window |
| `pulse projects` | Print project health summary |
| `pulse projects --repo pulse` | Filter to a specific repo |
| `pulse config init` | Generate default config file |
| `pulse config show` | Print current config |
| `pulse config adapters` | Show adapter status and missing env vars |
| `pulse version` | Print version |

All stdout commands support `--json` for scripting and `--verbose` for debug output.

## What It Collects

### Must-Have Collectors

- **Git Scanner** — branch state, dirty files, ahead/behind, last commit, stale branches across all your repos
- **GitHub Notifications** — PRs, issues, CI failures via the GitHub API
- **System Resources** — CPU, RAM, disk usage from `/proc` and `syscall`
- **Docker Status** — running containers, images, port mappings

### Cost Collectors (Stubs)

These are registered but awaiting billing API verification:

- **Claude** (Anthropic) — requires `ANTHROPIC_API_KEY`
- **Voyage AI** — requires `VOYAGE_API_KEY`
- **Tavily** — requires `TAVILY_API_KEY`
- **ElevenLabs** — requires `ELEVENLABS_API_KEY`

## Architecture

```
Collectors → Sync Engine → SQLite → Briefing Engine → Writers
```

- **`pulse sync`** (cron, hourly) runs all enabled Collectors, writes to SQLite
- **`pulse`** / **`pulse tui`** / **`pulse obsidian`** reads from SQLite, builds a Briefing struct, dispatches to a Writer
- No network calls during read commands — everything is instant

```
pulse/
├── cmd/pulse/main.go              # Entry point
├── internal/

│   └── tui/                       # Bubble Tea TUI (3 tabs, drill-down, keyboard nav)

```

## Configuration

Config lives at `~/.config/pulse/config.yaml`. Generated by `pulse config init`:

```yaml
projects:
  scan:
    - ~/projects
  ignore:
    - vendor
    - node_modules

github:
  username: your-github-username

obsidian:
  vault_path: ~/your-vault
  daily_note_path: "Daily Notes/YYYY-MM-DD.md"
  section_heading: "## Pulse Briefing"

adapters:
  git: true
  github: true
  claude: true
  docker: true
  system: true

sync:
  timeout: 30s

costs:
  default_period: 30d
  currency: USD
```

**Secrets**: API keys are read from environment variables only — never stored in config. Set them in your shell profile:

```bash
export GITHUB_TOKEN="ghp_..."
export ANTHROPIC_API_KEY="sk-ant-..."
```

Run `pulse config adapters` to see which env vars are set or missing.

## Project Discovery

Pulse auto-discovers git repos under your configured `scan` directories:

- **Max depth: 2 levels** — finds `scan_dir/project/.git` and `scan_dir/org/project/.git`
- **Default exclusions**: `node_modules`, `vendor`, `.cache`, `__pycache__`
- **Custom ignore**: add directory names to the `ignore` list in config

## TUI

Launch with `pulse tui` for an interactive dashboard:

| Tab | Key | Content |
|-----|-----|---------|
| Briefing | `1` | Morning briefing with projects, notifications, costs, system |
| Projects | `2` | All repos with status. `Enter` to drill into branches and commits |
| Costs | `3` | Per-service totals with bar charts. `Enter` for details |

**Navigation**: `j`/`k` scroll, `Enter` drill-down, `Esc` back, `q` quit

## Cron Setup

Pulse is designed to collect data on a schedule. Add to your crontab:

```bash
# Collect data every hour
0 * * * * pulse sync

# Or install via pulse (coming soon)
# pulse config init --cron
```

On WSL2, you may need to start the cron service:

```bash
sudo service cron start
```

## Data Storage

All data lives in `~/.config/pulse/pulse.db` (SQLite with WAL mode). Tables are linked by `sync_id` for time-travel queries across sync runs.

- **Briefing history**: 30-day retention, pruned during `pulse sync`
- **Collector data**: retained indefinitely

## Exit Codes

| Command | 0 | 1 | 2 |
|---------|---|---|---|
| `pulse sync` | All collectors succeeded | Partial — some failed | Total failure |
| Read commands | Success | Error | — |
