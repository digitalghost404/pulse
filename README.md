# Pulse

Personal command center CLI that answers **"what should I pay attention to right now?"**

Pulse combines morning briefings, project health monitoring, and AI service cost tracking into a single Go binary. Data collection runs via cron; reads are instant from a local SQLite database.

## Quick Start

```bash
# Install
go install github.com/xcoleman/pulse/cmd/pulse@latest

# Generate config
pulse config init

# Edit config to set your project scan directories
# ~/.config/pulse/config.yaml

# Collect data
pulse sync

# View your briefing
pulse
```

## Commands

| Command | Description |
|---------|-------------|
| `pulse` | Print morning briefing to stdout |
| `pulse sync` | Collect data from all enabled sources |
| `pulse sync --only git` | Run a single collector |
| `pulse tui` | Launch interactive Bubble Tea dashboard |
| `pulse obsidian` | Append briefing to today's Obsidian daily note |
| `pulse costs` | Print cost summary |
| `pulse costs --service claude` | Filter costs to a specific service |
| `pulse costs --period 7d` | Change the cost lookback window |
| `pulse projects` | Print project health summary |
| `pulse projects --repo pulse` | Filter to a specific repo |
| `pulse config init` | Generate default config file |
| `pulse config show` | Print current config |
| `pulse config adapters` | Show adapter status and missing env vars |
| `pulse version` | Print version |

**Global flags**: `--json` for scripting, `--verbose` for debug output, `--since <duration>` to override time window (e.g., `--since 24h`, `--since 7d`).

## What It Collects

### Core Collectors

- **Git Scanner** — branch state, dirty files, ahead/behind, last commit across all your repos
- **GitHub Notifications** — PRs, issues, CI failures via the GitHub API (requires `GITHUB_TOKEN`)
- **System Resources** — CPU (load average), RAM, disk usage from `/proc` and `syscall`
- **Docker Status** — running containers, images, port mappings, CPU/memory stats

### Cost Collectors

- **Claude** — parses local Claude Code JSONL logs at `~/.claude/projects/`. No API key needed. Supports two modes:
  - **API pricing** (default): estimates cost from per-model token rates
  - **Max subscription**: set `claude.subscription: "max"` and `claude.monthly_cost_cents: 10000` for fixed $100/mo proration
- **Tavily** — fetches usage from Tavily API, computes daily delta from cumulative counters. Requires `TAVILY_API_KEY`
- **ElevenLabs** — fetches character usage and subscription info. Requires `ELEVENLABS_API_KEY`
- **Subscriptions** — tracks fixed monthly costs (any service) configured in `costs.subscriptions`

### Stubs (Not Yet Implemented)

- **Voyage AI** — registered but awaiting billing API verification. Requires `VOYAGE_API_KEY`

## Architecture

```
Collectors → Sync Engine → SQLite → Briefing Engine → Writers
```

- **`pulse sync`** (cron, hourly) runs all enabled Collectors sequentially, each with its own timeout. Writes to SQLite. Failures in one collector don't abort others.
- **`pulse`** / **`pulse tui`** / **`pulse obsidian`** reads from SQLite, builds a Briefing struct, dispatches to a Writer
- No network calls during read commands — everything is instant
- Briefing history auto-pruned to 30 days during each sync

## Configuration

Config lives at `~/.config/pulse/config.yaml`. Generated by `pulse config init`:

```yaml
projects:
  scan:
    - ~/projects
  ignore: []

github:
  username: "your-github-username"

obsidian:
  vault_path: ""
  daily_note_path: "Daily Notes/YYYY-MM-DD.md"
  section_heading: "## Pulse Briefing"

# Claude Code cost tracking (reads local logs, no API key needed)
# claude:
#   subscription: ""          # "max" for fixed monthly, "" for API pricing
#   monthly_cost_cents: 0     # e.g., 10000 = $100/mo for Max plan

adapters:
  git: true
  github: true
  claude: true
  tavily: true
  elevenlabs: true
  docker: true
  system: true

sync:
  timeout: 30s

costs:
  default_period: 30d
  currency: USD
  pricing:
    tavily_cents_per_request: 1        # $0.01/request (pay-as-you-go)
    elevenlabs_cents_per_1k_chars: 30  # $0.30/1k characters
  # subscriptions:
  #   - name: "Example Service"
  #     service: "example"
  #     monthly_cost_cents: 999       # $9.99/mo
```

### Environment Variables

API keys are read from environment variables only — never stored in config:

| Variable | Collector | Required |
|----------|-----------|----------|
| `GITHUB_TOKEN` | GitHub notifications | Yes, for GitHub |
| `TAVILY_API_KEY` | Tavily usage/costs | Yes, for Tavily |
| `ELEVENLABS_API_KEY` | ElevenLabs usage/costs | Yes, for ElevenLabs |
| `VOYAGE_API_KEY` | Voyage (stub) | Not yet used |

Note: The Claude collector reads local log files — no API key needed.

Run `pulse config adapters` to see which env vars are set or missing.

### Cost Calculation

- **API-priced services** (Claude default): cost = tokens × per-model rate from built-in pricing table
- **Subscriptions** (Claude Max, custom): monthly cost ÷ 30 = daily prorated cost
- **Usage-priced services** (Tavily, ElevenLabs): usage × configured per-unit price from `costs.pricing`
- **Burn rate**: total cost ÷ actual days of data (not the configured period)

## Project Discovery

Pulse auto-discovers git repos under your configured `scan` directories:

- **Max depth: 2 levels** — finds `scan_dir/project/.git` and `scan_dir/org/project/.git`
- **Default exclusions**: `node_modules`, `vendor`, `.cache`, `__pycache__`
- **Custom ignore**: add directory names to the `ignore` list in config

## TUI

Launch with `pulse tui` for an interactive dashboard:

| Tab | Key | Content |
|-----|-----|---------|
| Briefing | `1` | Morning briefing with projects, notifications, costs, system |
| Projects | `2` | All repos with status. `Enter` to drill into branches and commits |
| Costs | `3` | Per-service totals with bar charts. `Enter` for details |

**Navigation**: `j`/`k` scroll, `Enter` drill-down, `Esc` back, `q` quit

## Cron Setup

Pulse is designed to collect data on a schedule. Add to your crontab:

```bash
# Collect data every hour
0 * * * * pulse sync
```

On WSL2, you may need to start the cron service:

```bash
sudo service cron start
```

## Data Storage

All data lives in `~/.config/pulse/pulse.db` (SQLite with WAL mode). Tables are linked by `sync_id` for time-travel queries across sync runs.

- **Briefing history**: 30-day retention, pruned during `pulse sync`
- **Collector data**: retained indefinitely
- **File permissions**: database and config created with `0600`/`0700` for security

## Exit Codes

| Command | 0 | 1 | 2 |
|---------|---|---|---|
| `pulse sync` | All collectors succeeded | Partial — some failed | Total failure |
| Read commands | Success | Error | — |
